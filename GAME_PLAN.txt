# SAVE THE BABY

## Master Development Plan (Version 1.0)

### Vision

Save The Baby is a story-driven Match-3 puzzle adventure inspired by the satisfying gameplay of Candy Crush Saga and Royal Match, while introducing an original rescue story, investigation mechanics, and cinematic storytelling.

The goal is to create a professional-quality mobile and desktop game that is scalable, data-driven, and easy to expand with additional levels and content without rewriting core code.

---

# Development Philosophy

This project follows several core principles.

## 1. No Hardcoding

Anything that can be stored as data must be stored as data.

Examples include:

* Levels
* Objectives
* Rewards
* Story
* Dialogues
* Characters
* Obstacles
* Powerups
* Animations
* Game settings

All of these should be loaded from JSON files or configurable resources whenever practical.

Adding Level 501 should require creating only a new JSON file, not modifying the engine.

---

## 2. Modular Architecture

Every system should be independent.

Example managers include:

GameManager
BoardManager
LevelManager
StoryManager
SaveManager
AudioManager
RewardManager
UIManager
AnimationManager
AdsManager

Each manager should have one clear responsibility.

---

## 3. Reusable Components

Instead of writing separate scripts for every object, reusable scripts will power multiple game elements.

Example:

One Tile.gd script should represent all collectible tiles.

The tile type will be determined by configuration rather than unique scripts.

---

## 4. Data-Driven Levels

Every level will be described using JSON.

Example properties:

* board size
* moves
* objectives
* tile types
* obstacles
* difficulty
* rewards
* tutorial messages

No level-specific code should exist.

---

## 5. Scalable Design

Every new feature should be designed so future content can be added without modifying existing systems.

# Long-Term Vision

Version 1.0:
A polished story-driven Match-3 game with approximately 50 handcrafted levels, cinematic storytelling, rewards, powerups, obstacles, and a complete rescue campaign.

Future updates may include:

* Additional chapters
* New villains
* More environments
* Seasonal events
* More powerups
* More obstacles
* Daily challenges
* Online leaderboards
* Cloud saves
* Achievement system
* Cosmetic themes
* Additional languages
* Expanded story content

The project should be designed from the beginning so these additions can be integrated with minimal changes to the core engine.



# Final Goal

Build a maintainable, professional-quality Match-3 engine that powers Save The Baby through reusable systems, configurable data, and scalable architecture. Every gameplay feature, story event, reward, and level should be easy to extend without rewriting existing code, allowing the game to grow from a solid first release into a long-term project.
